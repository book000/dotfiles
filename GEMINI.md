# Gemini CLI 作業方針

## 目的
Gemini CLI 向けのコンテキストと作業方針を定義します。

## 出力スタイル
- 日本語で簡潔に回答する。
- 重要な判断点と変更点を明示する。
- 箇条書きを活用し、視認性を高める。

## 共通ルール
- 会話は日本語で行う。
- コミットメッセージは Conventional Commits に従う（description は日本語）。
- 日本語と英数字の間には半角スペースを挿入する。

## プロジェクト概要
- 目的: chezmoi で dotfiles と AI エージェント設定を管理する。
- 主な機能: シェル設定、tmux 設定、通知設定のテンプレート化。

## コーディング規約
- フォーマット: 既存ファイルのインデントや改行ルールを維持する。
- 命名規則: chezmoi の命名規則 (`dot_`, `encrypted_`, `.tmpl`) に従う。
- コメント: 日本語で記載する。
- エラーメッセージ: 英語で記載する。

## 開発コマンド
このプロジェクトには `package.json` は存在しません。主な操作は `chezmoi` コマンドで行います。
```bash
# 設定の適用（確認）
chezmoi apply --dry-run

# 設定の適用
chezmoi apply

# ファイルの追加
chezmoi add <path>
```

## 注意事項
- 認証情報や機密情報を平文でコミットしたり、ログに出力したりしない。
- 既存のプロジェクトルールを優先し、独断で大規模なリファクタリングを行わない。
- Gemini CLI の除外設定を確認し、不要なファイルを読み込まないようにする。

## リポジトリ固有
- `home/` 配下が chezmoi のソースであり、ここでの変更が最終的な成果物となる。
- エージェント用プロンプトは、ルート (`CLAUDE.md`, `GEMINI.md` 等) と `home/` 配下の両方に存在するが、ルートのものはこのリポジトリ自体の開発用である。
- age による暗号化が設定されているため、機密情報の追加時は `chezmoi add --encrypt` を使用する。
